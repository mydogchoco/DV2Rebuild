## A. 공통 / 용어 (에셋 strings에서 확인된 것)

- 화폐: **다이아(diamond)**, **골드(gold)**, **샌즈(senz)**  🟦 확정(존재)
  - 각 화폐의 용도/획득처/오프라인에서의 위상은? 
  다이아(diamond): 현금결제로 얻는 캐쉬 재화. 게임사의 수입원 중 하나였으나, 이제 수익을 낼 필요 없으므로 수급을 풀어도 됨. 주로 "다이아 상점"이나 "알 상점",  "방랑상인" 등 여러 상점에서 특수 아이템들을 구매할 때 쓰이거나, 용 레벨업 수치 재설정 등에 폭넒게 쓰임
  골드(gold): 인게임 내에서 탐험이나 아이템 판매 등으로 얻을 수 있는 재화. 상점이나 경매장에서 아이템을 구매할 때 쓰인다. 
  샌즈(senz): 다이아와 골드 외에 샌즈라는 이름의 주요 재화는 존재하지 않았음. "샌즈의~" 수식어가 붙는 젬 관련 아이템이나 기타 재료와 관련된 게 아닐까? 
- 능력치: 공격력(att), 방어력(def), 생명력(hp), 치명타(cri), 회피율(evd), 방어율(blkrate)  🟦 확정(존재)
- 드래곤 **전투 유형 6종** 🟦 확정(존재):
  공격형(atk) / 체력형(hp) / 방어형(def) / 체공형(ha) / 공방형(ad) / 체방형(hd)
  - 각 유형이 스탯 분배/플레이에 주는 정확한 차이는? (설명문 일부는 strings에 있음)
  치명타(cri), 회피율(evd), 방어율(blkrate)는 용마다 기본 수치가 일괄 10%로 설정되어 있으며, 레벨업으로는 올릴 수 없음.
  레벨업을 통해 오르는 스탯은 생명력/공격력/방어력 3가지이며, 드래곤의 전투 유형에 따라 초기 스탯 분배와 레벨업 시 최대 스탯 상승량이 다름.

  > 🔎 Claude 보완(육성팁 문서) — 레벨업 외 능력치 보정 수단도 존재: **문장 강화**(체/공/방 각 최대 20레벨, 레벨당 +2%; 5레벨 이상 시 **각인** 해금), **각성기 게이지**(필살기 성격의 각성기를 채워 발동) 등. 이들은 §C·§D 전투 계산에 영향.
- **세대(generation)**: 소수점 값(`%.1f`)으로 표시됨 🟦 확정(존재)
  - 세대의 의미와 게임 규칙상 영향은?
  세대가 높아질수록 초기 스탯과 레벨업 시 스탯 상승량이 더 높은 드래곤이다. 간단히 말해서 세대가 높은 드래곤이 더 좋다.
  보통 세대가 높은 드래곤일수록 나중에 출시되었기에 이름을 "세대(generation)"로 지은 것 같다.
  연속적으로 존재하지 않고 2.4세대 -> 3세대 처럼 gap이 존재하는 것으로 알고 있는데, 이는 정상이다.
  인게임에서 용마다 성급이 존재하며, 이는 레어도의 척도라고 볼 수 있다. 2.4세대까지는 4성, 그 이상부터는 5성이나 6성으로 기억하고 있다.

  > 🔎 Claude 검토(추가 위키 근거: 육성팁·레이드·맵 문서) — **"세대"와 "등급"은 별개 개념**입니다. 위키 전반은 용에 표시되는 소수점 값(`%.1f`)을 **등급(grade)**으로 부릅니다. 정리하면:
  > - **세대(generation)** = 종(species) 단위의 출시 계보(1세대 / 2.1 / 2.3 / 2.4 / 3 / 3.4 / 4세대 / 이벤트 / 스페셜). 비연속(2.4→3 gap 정상)이며 높을수록 일반적으로 강함 — 여기까지는 사용자 서술과 일치. 포포 포인트 상점에서 "세대 선택권"으로 판매됨.
  > - **성급(star)** = 레어도(4성/5성/6성). 가챠 확률 예: 4성 2.97% / 5성 0.89% / 6성 0.3%. 세대와 상관관계는 있으나 별개 축(위키: "별 개수가 같으면 세대·종류 무관하게 스프라이트 공유").
  > - **등급(grade)** = 개체(인스턴스)별 소수점 품질값. 부화 시 기본값(예 8.0)에서 시작해 육성으로 올라가며 10.0+도 가능. 상승 요소: 알 강화(3강/4강), 축복받은 둥지(+0.6), 포텐셜(1레벨 리셋), 아모르의 축복(3개 +0.9), 데르사의 축복(하락 방지), 각성(+0.6), 초월된 아모르(0.1×6=+0.6), 한계돌파.
  > ⇒ 사용자 서술의 "소수점 값으로 표시되는 것"은 (에셋 라벨이 generation이더라도) **실질적으로 등급(grade)일 가능성이 높음**. 데이터 설계 시 세대(종 속성)·성급(레어도)·등급(개체 런타임 값)을 분리 권장(§B 스키마 메모 참고). 에셋 문자열 라벨은 한 번 확인 부탁.

## B. 드래곤 (수집 / 도감 / 성장 / 진화)

**에셋 근거:** `dragon_N_baby/child/adult_spine`(성장 3단계), `_critical/_e/_e_critical`(변형).

- 속성(elements) 목록과 상성표는? (에셋 aura: aqua/chaos/dark/earth/fire/holy/light/shadow/wind 확인됨)
aqua는 fire에 유리하고 earth에 불리하다
earth는 aqua에 유리하고 wind에 불리하다
fire는 wind에 유리하고 aqua에 불리하다
wind는 earth에 유리하고 fire에 불리하다
light는 dark에 유리하고 aqua, earth, fire, wind에 불리하다
dark는 aqua, earth, fire, wind에 유리하고 light에 불리하다
holy와 chaos는 shadow를 제외한 모든 속성에 유리하며, 서로의 약점을 찌른다.
shadow는 모든 속성에 대해 무상성이다.
유리한 속성은 1.5배의 데미지 보정을 받는다. (※ 1.5는 원작 관찰값. 자체 밸런스로 §K-3에서 유리 1.25 / 불리 0.85로 조정 채택 — 본 게임 적용값은 K-3 기준.)

> 🔎 Claude 검토 — 위 상성표는 추가 위키와 모순 없음(확인된 단서: 속성 9종 = 4원소+빛/어둠+신성/혼돈+무상성 그림자, "상성 존재", 땅·불은 상성 관계 없음 — 사용자 표와 일치). 다만 위키엔 쌍별 배수 전체 표/이미지는 텍스트로 안 잡혀 사용자 관찰을 기준값으로 사용.
- 성장 단계: baby → child → adult 전환 조건은? (레벨? 아이템? 시간?)
드래곤은 알에서 부화하면 1레벨에서 시작하며, 최대 45레벨까지 오를 수 있다.
9레벨까지는 baby, 10~19레벨은 child, 20~45레벨은 adult이다. 
레벨업은 탐험을 통해 경험치를 쌓거나, 레벨업 아이템을 사용해 진행할 수 있다.

> 🔎 Claude 검토(육성팁·던전 문서) — **레벨 상한 보정 필요**: 일반 상한은 45가 맞지만, **각성 후 50레벨까지** 성장 가능(위키: "50레벨을 찍고 각성", "45~50레벨 성체 드래곤", "초월된 아모르의 축복은 레벨 50이 된 용에게"). ⇒ adult 단계는 사실상 **20~만렙(45, 각성 시 50)**. baby/child/adult 전환 레벨(9/10/20)은 사용자 관찰값 그대로 둠(위키 반증 없음).
- **진화(캡슐 진화 포함)** 규칙: 무엇이 무엇으로, 조건/재료는?
45레벨 달성 이후 '각성의 마석'과 '아니마', '보네르' 아이템을 이용해 원하는 드래곤을 각성시킬 수 있다.
이때 드래곤의 성급(세대)별로 필요한 재화의 수가 다르다. 각 드래곤은 각성 시 지정된 '각성 스킬'을 얻으며, 추가로 등급을 올릴 수 있게 된다.

> 🔎 Claude 검토/보완(육성팁·아이템 문서) — 재료 '각성의 마석/아니마/보네르'는 위키와 일치(각성의 마석 = "n성 드래곤을 각성시키는 마석", 아니마·보네르는 검은섬에서 수급). 보정 1: 필요량은 **성급(별 4/5/6성) 기준**으로 다름(괄호의 "세대"보다 "성급"이 정확). 보정 2: 각성 효과 = **등급 +0.6**, 레벨 상한 45→50 해금, 각성 스킬 부여, 각성기 게이지/장비 아티팩트 연계. (참고: 각성 후 '초월된 아모르의 축복' 사용 가능.)
- 레벨 성장: 레벨1 기본 능력치 + 레벨업당 상승치 모델은? (strings에 "최대 상승치" 개념 존재)
드래곤의 세대와 전투 유형별로 모델이 다르다. 별도의 표로 기입 예정.

  > 🔎 Claude 검토(`docs/DragonStat.xlsx` 수령·디코딩 완료) — base/growth는 **개체별이 아니라 `(전투유형 6종 × 성급/세대 티어)`로 파라미터화**돼 있습니다(값 순서 = 체/공/방). 티어 열: 2성(스타터)·3성(1~1.9세대)·4성(2~2.3세대)·5성(2.4~3.3세대)·6성(3.4~3.9세대)·6성(4세대)·커스텀.
  > ⇒ 데이터 설계 시 `dragons.json`엔 per-dragon `base_stats`/`growth`를 넣지 말고 **`type`+`star_tier`만 두고 이 표를 룩업**하게 한다(아래 스키마 메모 갱신). 전체 수치 출처 = `docs/DragonStat.xlsx`(Phase 2에서 그대로 데이터화). 적용식은 §K-1에서 확정.

> 🔎 Claude 보완 — 위 "레벨업당 상승치"는 종(species)의 **고정 성장 모델**(base+growth, §B 스키마)이고, 이와 별개로 개체별 **등급(grade) 보정 시스템**이 최종 능력치에 곱/가산됨: 포텐셜(1레벨 스탯 리셋), 축복류(드래곤/마이아/데르사/아모르/초월된 아모르), 각성(+0.6), 문장 강화(체·공·방 2%/lv, 최대 20), 한계돌파. ⇒ 정적 데이터(종 성장표)와 런타임 데이터(개체 등급/문장/각인)를 분리 권장.
- 도감 분류 기준(세대/속성/등급 등)?
기본적으로 속성과 등급, 전투 유형으로 구분 가능하며, 세대는 인게임 내에서 간접적으로 확인 가능하다.

**데이터 표현 형식(목표 — Claude가 채울 스키마 예시):**

> 🔎 Claude 보완 — `generation`(종 계보)과 `star`(레어도)는 **정적 종 데이터**, `grade`(개체 소수점 등급)·문장·각인은 **세이브별 런타임 데이터**로 분리. 아래 스키마에 `star` 추가.

```jsonc
// data/dragons.json — 한 항목 예시(종 단위 정적 데이터; 값은 사용자 복원분으로 채움)

  "name_key": "dragon_1_name",        // strings 키
  "element": "fire",                   // 속성
  "type": "atk",                       // 전투 유형 6종 중

  "star_tier":    "4star",            // base/growth 룩업 키 (DragonStat.xlsx 티어) ← 갱신
  // base_stats/growth는 per-dragon이 아니라 (type × star_tier) 표(DragonStat.xlsx)에서 룩업 ← 갱신
  "max_level":    45,                  // 일반 45 / 각성 시 50  ← 추가
  "stages":       { "baby": "dragon_1_baby_spine", "child": "dragon_1_child_spine", "adult": "dragon_1_adult_spine" },
  "awaken":       { "to": null, "skill": null, "grade_bonus": 0.6 },     // 각성 대상/스킬/등급보정  ← 추가
  "skills":       []                   // 보유 스킬 id 목록
}
// 개체별 런타임 값(별도 세이브): grade(소수점 등급), crest{hp,att,def}(문장 lv), seal(각인), potential 등
```

## C. 스킬

**에셋 근거:** 스킬 아이콘 `skill/{id}.png` (ID 매핑 확정).

- 스킬 분류(액티브/패시브, 단일/광역, 속성 등)?
  [출처: 나무위키/스킬]
  슬롯/스킬 타입은 △·□·○ 3종 + ☆(만능) 4종. 각 스킬에 타입이 지정되어 있고, 장착하려면 슬롯 타입과 일치해야 한다(☆ 슬롯은 모든 타입 수용). 다이즈의 호신부로 슬롯 타입을 랜덤 변경하며 낮은 확률로 ☆ 슬롯이 나온다.
  스킬 슬롯은 최대 2칸이며 레벨 10·35 달성 시 해금된다.
  작동 방식별 분류(관찰 기준): 즉발 공격형(심판의 날개, 거신의 돌격 등), 방어/피해감소형(철갑방패, 바위방어술, 보호의 장막), 버프형(돌격지시, 맹수의 감각, 신속이동), 디버프/상태이상형(빙결의 표식=회피0, 뼈부수기=방어·회피불가, 마비의 구름=행동불가, 시간의 역행=크리·회피·방어0), 지연/설치형(시한폭탄: 7턴 후 발동), 자해유도형(환각효과), 회복/정화형(치유의 빛, 빛의 정화).
  패시브로 전환된 스킬도 있다(예: 필살방어는 패치 후 패시브화).
  일부 스킬은 속성이 부여되어 있어(야수의 본능, 돌격지시 등) 습득·레벨업 시 드래곤 속성과 일치해야 한다.
  단일/광역: 3vs3에서 "타입 일치 시 멤버 전체 적용"되는 스킬이 언급됨(야수의 본능 등). 그 외는 단일 대상이 기본. (스킬별 단일/광역 플래그를 정리한 표는 제공 파일에 없음 → 개별 플래그는 비움)
- 위력/계수/쿨다운/소모치 결정 방식은?
  [출처: 나무위키/스킬]
  스킬은 레벨 1~5이며 레벨이 높을수록 효과가 커진다. 레벨1을 배워야 레벨2 이상을 습득할 수 있다.
  효과는 스킬 레벨 기반 수식으로 표기된다. 확인된 예시:
  · 철갑방패: 피해 감소 = 10 + 스킬레벨×5
  · 바위방어술: 피해 감소 = 25% + 스킬레벨×5%
  · 필살방어: 방어력 증가 = 20 + 스킬레벨×5 (+ 1회 한정 체력1 생존)
  · 시한폭탄: 7턴 후 자신 체력의 (15 + 스킬레벨×2)% 피해, 전용 스킬이면 (20 + 스킬레벨×2)%
  · 돌격지시: 공·방 +7% + 스킬레벨×3%
  · 맹수의 감각: 크리 +4%(전용 7%)+SL, 공격력 +5%(전용 10%)
  · 신속이동: 회피 +7%(전용 9%)+SL
  · 고속이동: 크리 +3%(전용 4%)+SL, 회피 +4%(전용 5%)+SL, 공격력 +3%(전용 6%)
  "쿨다운" 개념보다는 ① 발동 확률(proc) 기반, ② 지속(턴) 스킬(효과를 받은 드래곤이 공격할 때마다 지속 턴 1 차감), ③ 지연 발동(시한폭탄: 7턴 후), ④ 일정 체력 이하 조건 발동(신의 분노 등)으로 운용된다.
  전용 스킬칸에 넣으면 강화 보정(전용 수치) 적용. 아티팩트(이그니스/마리스/벤투스/루멘/옵스큐럼/테라)가 스킬 효과 강화·발동 조건 완화·발동 확률 증감에 관여.
  소모치(마나 등) 개념: 제공 파일에서 확인 안 됨 → 비움.
- 드래곤–스킬 연결 규칙(레벨업 습득? 고정?)?
  [출처: 나무위키/스킬]

- 가챠 확률(원작 일부: 4성 2.97% / 5성 0.89% / 6성 0.3%) 그대로 / 자체 조정 → 가챠 확률은 유지. 가챠 재화를 넉넉히 지급하는 방식으로 간접 완화 계획.

### K-9. 데이터 표준화 확인 (Claude 작업용)
- 속성 내부 키는 영문(aqua/earth/fire/wind/light/dark/holy/chaos/shadow)으로 통일 → ✅ 채택
- 전투 유형 내부 키: atk/hp/def/ha(체공)/ad(공방)/hd(체방) → ✅ 채택
