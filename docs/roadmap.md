> 이 문서는 "무엇을 / 어떤 순서로 / 어떻게" 할지의 단일 기준이다.
> CLAUDE.md(절대 규칙)와 짝을 이루며, 단계가 끝날 때마다 체크박스를 갱신한다.
> 작성일 기준: 2026-06-23.

## 0. 한 줄 요약

서비스 종료된 온라인 모바일 RPG의 **그래픽/오디오 에셋만 남은 상태**에서,
**유실된 게임플레이 설계를 사용자가 복원**하고 → **Claude가 그걸 데이터/코드로 구조화**해 →
**Godot 4 싱글플레이 오프라인 게임**으로 재구현한다.

## 1. 프로젝트를 규정하는 3가지 사실 (반드시 기억)

1. **에셋에는 게임플레이 데이터가 없다.** DV2/ 레포는 그래픽(spine/plist/png/jpg),
   오디오(mp3), 파티클, 셰이더, 폰트, UI 문자열(xml)뿐. 드래곤 스탯·스킬 수치·
   스테이지·부화/조합·전투공식·성장곡선은 **전부 서버에 있었고 사라졌다.**
   → 이 부분은 **사용자가 웹서핑 + 자체 밸런스로 복원**한다. (역할 분담의 핵심)
2. **비주얼은 거의 다 있다.** 드래곤은 baby/child/adult 성장 3단계 + 속성/크리티컬
   변형이 spine 키별로 존재. 스킬 아이콘은 `skill/{id}.png`로 ID 매핑. 즉
   **설계 스켈레톤만 만들면 살을 붙일 그래픽은 대부분 확보돼 있다.**
   (단서: ① 옛 Spine 포맷 로드 가능 여부 ② ID별 에셋 커버리지는 Phase 0에서 확인)
3. **오프라인 전용으로 변형한다.** 온라인 시스템(친구/PvP 콜로세움/경매·거래/랭킹/
   서버이벤트/로그인)은 **삭제**. 원래 멀티였던 콘텐츠(레이드 등)는 **솔로 플레이
   가능하도록 밸런스·방식을 새로 설계**한다.

## 2. 역할 분담

| 영역 | 담당 | 비고 |
|---|---|---|
| 유실된 게임 시스템 지식 제공 (스탯/공식/규칙) | **사용자** | 평문/위키 출처로 |
| 그 지식을 설계문서·데이터 스키마로 구조화 | **Claude** | 아래 §4 워크플로 |
| 코드/에셋 파이프라인/엔진 구현 | **Claude** | 사용자는 검수·플레이 |
| 밸런스 결정, 오프라인 변형 방향 | **사용자** (Claude 제안) | |
| 게임 개발 순서·툴 사용법 안내 | **Claude** | 사용자는 비-게임개발자 |

> 사용자는 기본 프로그래밍 지식만 있고 게임 개발 순서·툴은 익숙하지 않다.
> 따라서 Claude는 **"무엇을 어떤 형식으로 알려주면 되는지"를 항상 먼저 제시**한다.
> 사용자가 형식을 몰라 막히지 않도록, 빈 칸을 채우는 방식(템플릿)으로 진행한다.

## 3. 도구 환경

- **Python**(정식 설치 필요): 일회성 변환/추출 도구 전부. plist는 표준 `plistlib`.
- **Godot 4**(설치 필요): 런타임 엔진.
- Node 불필요. 현재 머신엔 셋 다 미설치 → Phase 0에서 설치.
- ⚠️ **Spine 리스크**: spine_json이 구버전 포맷(≈2.x/3.x). 최신 spine-godot가
  그대로 못 읽을 수 있음 → Phase 0에서 **로딩 스파이크 먼저**.

## 4. 지식 → 게임으로 바꾸는 워크플로 (사용자가 막히지 않게)

게임 시스템 하나를 다룰 때 매번 이 4단계를 반복한다:

1. **인터뷰**: Claude가 그 시스템에 대해 채워야 할 항목을 질문지로 제시.
   (예: "부화 시스템 — 알의 종류는? 부화 시간 결정 방식은? 조합 규칙은?")
2. **설계문서화**: 사용자의 평문 답변을 `docs/game_design.md`의 해당 섹션에
   규칙으로 정리. 출처/확정여부 표시. 미확정은 TODO + `docs/reverse_engineering.md`.
3. **데이터 스키마화**: 수치가 들어갈 표 형태(JSON 스키마)를 `data/`에 정의하고,
   사용자는 **값만 채운다**. (필드가 뭘 의미하는지 Claude가 주석/스펙으로 설명)
4. **구현·검증**: 그 데이터를 읽어 동작하는 최소 단위를 만들고 화면/테스트로 확인.

> 즉 사용자는 "JSON을 어떻게 쓰지?"를 고민할 필요 없이, **질문에 답하고 표의 빈
> 칸을 채우기만** 하면 된다. 형식·구조·코드는 Claude가 책임진다.
